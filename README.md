# MyResearchClaw

MyResearchClaw is a `Codex`-oriented research workflow for finding, tracking, and deep-reading papers from top venues.

## What Codex Should Use

- Project instructions: `AGENTS.md`
- Project skill index: `SKILL.md`
- Search skill: `skills/conference-scout/`
- Reading skill: `skills/paper-reader/`
- Engineering skill: `skills/engineering-scout/`

The `skills/` directory is the authoritative source for Codex.

## Core Workflows

### `conference-scout`

Use when you want Codex to search top-conference papers by topic, year range, and venue group, then update the persistent kanban.

Example prompts:

```text
帮我搜索 wearable audio privacy 在 IoT 顶会 2022-2025 年的论文
Use conference-scout to find recent federated learning papers in NeurIPS, ICLR, and ICML since 2023.
```

### `paper-reader`

Use when you want Codex to deeply read one paper and generate a structured DNL note.

### `engineering-scout`

Use when you want Codex to search the same topic from the engineering side: GitHub implementations, products, and real-world deployment signals.

For broad topic investigation, run `conference-scout` and then `engineering-scout` on the same topic so the paper page and engineering page stay aligned.

Example prompts:

```text
帮我用 paper-reader 精读这篇论文：https://arxiv.org/abs/2401.XXXXX
Read this paper with paper-reader: https://doi.org/10.1145/XXXXXXX
```

## Persistent Outputs

| Path | Purpose |
|---|---|
| `output/papers.json` | Persistent paper database across searches |
| `output/kanban.html` | Default entry page; single-topic kanban or multi-topic navigator |
| `output/projects/{topic_slug}/papers.html` | Topic-specific kanban page with `Read Paper` and `Engineering View` |
| `output/notes/` | Markdown reading notes generated by `paper-reader`, grouped by topic slug |
| `output/projects/{topic_slug}/engineering.html` | Topic-specific engineering implementation report page |

Paper lifecycle: `unread` -> `reading` -> `done`

## Runtime

The local reading server is Codex CLI-based:

```bash
cd /home/meng/Agent/MyResearchClaw
python serve.py
```

Optional environment variables:

- `MYRESEARCHCLAW_MODEL` defaults to `gpt-5.4-mini`
- `MYRESEARCHCLAW_CODEX_BIN` can point to a non-default `codex` binary path

Quick checks:

```bash
curl http://localhost:5678/api/health
curl http://localhost:5678/api/papers
```
